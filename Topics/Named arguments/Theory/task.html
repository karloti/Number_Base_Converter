When we invoke a function that has parameters, we can pass arguments by the names of these parameters. This approach can improve the readability of your code, especially if a function has lots of parameters. Also, it is a convenient way to change the order of arguments passed in a function call.

Improving code readability

Imagine a cashier who sells cinema tickets, and within a single day the price does not change. You need to calculate how much money will be in the cash desk at the end of the work day. Here is a function that has three parameters:

fun calcEndDayAmount(startAmount: Int, ticketPrice: Int, soldTickets: Int) =
        startAmount + ticketPrice * soldTickets
  • startAmount is a sum in the cash desk before starting this working day.
  • ticketPrice is the price of one ticket today.
  • soldTickets is the number of tickets sold on this working day.

Since it is a regular function, let's invoke it as we usually do.

val amount = calcEndDayAmount(1000, 10, 500)  // 6000

This works fine, although there is one problem: unclear arguments. Of course, we can declare well-named variables and pass them, but sometimes you have to work with literals, not variables. With named arguments, we can make the code much more readable. To fix this problem, you may name each argument of this function when calling it:

val amount = calcEndDayAmount(
    startAmount = 1000,
    ticketPrice = 10,
    soldTickets = 500
)  // 6000

Now, the code is easy to understand!

Reordering arguments

Using named arguments, you can change the order of the arguments in a function call. Just specify the names in any order needed:

val amount = calcEndDayAmount(
    ticketPrice = 10,
    soldTickets = 500,
    startAmount = 1000
)  // 6000

Mixing named and positional arguments

You can also call a function with named and regular (positional) arguments, as long as all the named arguments are placed after all the positional ones.

calcEndDayAmount(1000, ticketPrice = 10, soldTickets = 500)  // 6000

Since Kotlin 1.4 you may use the positional arguments after the named arguments, but you need to keep the order of arguments. For example: 

calcEndDayAmount(ticketPrice = 10, 500, 1000)   // OK
calcEndDayAmount(startAmount = 1000, 10, 500)   // Incorrect invocation!


calcEndDayAmount(soldTickets = 500, ticketPrice = 10, startAmount = 500) // OK
calcEndDayAmount(soldTickets = 500, ticketPrice = 10, 500)  // Incorrect invocation!

Default and named arguments

It is convenient to use named and default arguments together. As you probably know, with default arguments, sometimes Kotlin does not understand which parameters to assign.

Let's modify the previous function making the first parameter optional:

fun calcEndDayAmount(startAmount: Int = 0, ticketPrice: Int, soldTickets: Int) =
        startAmount + ticketPrice * soldTickets

Suppose we would like to invoke this function passing only the two last parameters, but it does not work:

val amount = calcEndDayAmount(10, 500)  // It does not work:
                                        // no value passed for soldTickets

Here 10 is assigned to the first optional argument startAmount, not to the second parameter ticketPrice.

To invoke this function as you want it, we must use named arguments:

val amount = calcEndDayAmount(ticketPrice = 10, soldTickets = 500)  // 5000

Now it works!

Named arguments and default values

The default values for the parameters of a function are not necessarily constant. They can also be another named argument or a function.

For example:

fun sum2(a: Int, b: Int = a) = a + b
 
sum2(1)    // 1 + 1
sum2(2, 3) // 2 + 3

The following function declaration is incorrect since the parameter b is uninitialized when used:

fun sum2(a: Int = b, b: Int) = a + b

Conclusion

To sum it all up, use named arguments, and use them wisely, not only to improve code readability but also change the order of arguments when some parameters have default values.
